My Flicks

This provides users with access to information aobut different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Business Logic URL HTTP Method Query Parameters Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None None A JSON holding data about all the movies
Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user /movies/[title] GET Title None A JSON object holding data about a single movie, containing a description, genre, director, image URL, and if it's featured. Ex:
{
title: ,
genre: ,
director: ,
description: ,
image URL: ,
featured:
}
Return data about a genre (description) by name/title (e.g., "Thriller") /movies/genre/[title] GET Title None A JSON object holding data about a genre, containing a description.
Return data about a director (bio, birth year, death year) by name /movies/[director] GET Director None A JSON object holding data about a director, containing bio, birth year, and death year.
Allow new users to register /users/register POST None A text message indicating the user has been registered.
Allow users to update their user info (username) /users/[username]/change PUT Username None A text message indicating the username was updated
Allow users to add a movie to their list of favorites /movies/users/fav/[movie] POST Movie A JSON object holding data about the movie to add, structed like:
{
title: ,
genre: ,
director: ,
description: ,
image URL: ,
featured:
} 		A text message saying whether the movie has been added.
Allow users to remove a movie from their list of favorites /movies/[user]/fav/[movie] DELETE User, Movie None A text message saying whether the movie was successfully removed
Allow existing users to deregister /users/[username] DELETE username None A text message saying whether the user was successfully deregistered.
